# [PERF-003] AI 리포트 생성 60초 초과 시 "처리 중" 안내 UI 표시

## 1. 목적
- AI 리포트 생성에 최대 60초가 소요될 수 있으므로, 사용자가 페이지를 이탈하거나 오작동으로 인식하지 않도록 명확한 피드백을 제공한다.

## 2. 구현 범위
- 진단 폼 제출 후 AI 결과를 기다리는 대기 화면 또는 로딩 오버레이 구현
- 시간에 따른 로딩 텍스트 변화 또는 인터랙티브 애니메이션 적용

## 3. 입력/출력
### Input
- 폼 제출 이벤트 발생 및 로딩 상태 (isLoading)
- 타이머 상태 경과 시간

### Output
- 대기 화면 렌더링

## 4. 처리 로직
1. 진단 폼 제출 시 전체 화면 오버레이 혹은 로딩 페이지로 전환.
2. 스피너 컴포넌트와 함께 "AI가 브랜드 분석을 진행하고 있습니다..." 등의 문구 노출.
3. 로딩 상태가 특정 시간(예: 5초, 15초, 30초) 지속될 경우, 사용자가 지루하지 않게 팁이나 문구가 순차적으로 변경되도록 처리.
4. (Vercel 타임아웃 우려 시) 완료 응답을 받으면 성공 화면 혹은 검수 대기 안내 화면으로 이동.

## 5. 완료 기준
- [ ] 폼 제출 시 로딩 UI가 즉각적으로 표시되는지 확인
- [ ] 지연 시간이 발생해도 화면이 멈추지 않고 로딩 피드백이 지속되는지 확인

## 6. 관련 태스크
- 선행: FE-002, C-005
