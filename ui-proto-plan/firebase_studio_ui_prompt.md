# Role & Objective
당신은 최고 수준의 UI/UX 디자이너이자 프론트엔드 엔지니어입니다. 
지금부터 "5060 시니어를 위한 프리미엄 퍼스널 브랜드 매니지먼트" 서비스의 MVP 웹 화면들을 생성해야 합니다. 
기능 구현 이전에 완벽한 디자인과 사용자 경험(UX)을 갖춘 UI 뼈대를 만드는 것이 목표입니다.

# Tech Stack
- Framework: Next.js (App Router)
- Styling: Tailwind CSS
- Component Library: shadcn/ui, Radix UI
- Icons: Lucide React

# Design System & Vibe (매우 중요)
1. **Target Audience (5060세대):** 가독성이 핵심입니다. 글씨 크기는 큼직하게(최소 16px 이상), 텍스트와 배경의 대비를 명확하게 하세요.
2. **Premium Aesthetic:** 싸구려 느낌의 원색은 피하세요. 메인 컬러는 로열블루(#0123b4), 서브컬러는 그레이, 화이트를 사용해 주세요.  여백(Whitespace)을 과감하게 사용하여 우아함을 강조하세요. 
3. **Interactions:** 버튼 호버, 폼 포커스 시 부드러운 트랜지션(transition-all duration-300)을 적용하여 앱이 살아있는 듯한 느낌을 줍니다.
4. **Layout:** 모바일 반응형을 기본으로 하되, 데스크탑에서는 가로 폭을 넓게 쓰는 잡지(Magazine)나 고급 카탈로그 형태의 레이아웃을 사용하세요.

# Pages to Implement

아래 4가지 핵심 화면의 코드를 차례대로 작성해 주세요. 더미 데이터(Mock-up)를 넣어서 눈으로 바로 확인할 수 있게 해주세요.

## 1. Landing & CTA (Q-001)
- **역할:** 서비스 소개 및 진단 유도
- **UI 요구사항:** 
  - 신뢰감을 주는 고급스러운 Hero 섹션
  - 서비스의 가치(Value Proposition)를 텍스트와 아이콘으로 우아하게 배치
  - 화면 중앙과 하단에 돋보이는 "내 브랜드 진단 시작하기" CTA 버튼

## 2. 16문항 진단 폼 (Q-002, FE-001, DPR-001, PERF-003)
- **역할:** 고객이 16개의 질문에 답을 입력하는 화면
- **UI 요구사항:**
  - 질문 16개가 한 번에 나열되지 않고 스크롤에 따라 부드럽게 나타나거나, 진행률(Progress bar)이 보이는 카드 형태.
  - 각 질문에는 질문 텍스트와 작성 팁(`assetHint`)이 시인성 좋게 배치되어야 함.
  - 넓고 타이핑하기 편한 `textarea` 컴포넌트.
  - 하단에는 "개인정보 수집 및 AI 처리 동의" 체크박스 명시.
  - "제출하기" 버튼 클릭 시, 60초간 대기할 수 있는 세련된 "AI가 분석 중입니다..." 로딩 애니메이션 화면(오버레이) 구현.

## 3. 관리자 대시보드 및 리포트 편집기 (Q-003, Q-004, FE-003, FE-004)
- **역할:** 관리자가 인입된 요청을 보고, 리포트를 수정 및 승인하는 사내용 화면
- **UI 요구사항:**
  - **목록 화면:** 고객 이름, 연락처, 상태(대기, 완료 등), 날짜가 깔끔하게 보이는 테이블 형식.
  - **상세/편집 화면 (Split View):** 
    - 왼쪽 창: 고객이 입력한 16개 답변 내역 고정
    - 오른쪽 창: AI가 생성한 리포트 결과물(JSON 구조)을 각 섹션별로 텍스트로 수정할 수 있는 에디터 폼
  - 에디터 하단에 [리포트 승인], [거부], [AI 1회 재생성]을 할 수 있는 직관적인 버튼 그룹 배치.

## 4. 고객용 진단 리포트 웹뷰 (Q-005, FE-005, FE-006)
- **역할:** 관리자 승인이 완료된 후 고객이 받아보는 최종 결과물 화면
- **UI 요구사항:**
  - 퍼스널 브랜딩 결과를 보여주는 고급 매거진 스타일. 카드 뉴스처럼 섹션이 나누어져 있어야 함.
  - 미승인 상태일 때 보여줄 세련된 "리포트를 준비 중입니다" 빈 화면(Empty State) 포함.
  - 리포트 최하단에는 고객이 다음 액션을 취할 수 있도록 "1:1 코칭 신청하기" 프리미엄 CTA 버튼 고정(Sticky) 배치.
